Narrative

1. Artifact Description

This artifact is a further continuation of the same artifact that I used previously for Milestone 2, the Android weight-tracking application. I originally created this application in CS-360 about a year and a half ago. I improved the software engineering and design aspects for this same artifact in Milestone 2. This app is used to record the weight and calorie logs for a user over time, while emphasizing short-term success by rewarding stars for hitting little milestones. However, using SQLite, a user can see their data over a long period of time in a variety of ways.

2. Justification & Enhancements

This artifact shows real-world potential, as it is a very detailed recording of weight loss. By encouraging users to focus on short-term success, they will feel more encouraged to continue. This also encourages more frequent logging, which provides rich insights into body fluctuations. I selected this artifact for both the software design enhancement and the algorithms and data structure enhancement. For this, I added new data aggregations, iterations, lists, and visual charts.

3. Course Outcomes

The highlight of this enhancement is the new charting system, which can reflect a user’s data over daily, weekly, or monthly views. Users can toggle between line and bar graphs and explore entries from 2020–2026. In daily mode, every log entry is shown. In weekly and monthly modes, averages are calculated per weekday or per month. Additionally, the app now features dynamic goal weight adjustments, real-time star updates, and targeted data filtering for selected days. The goal of this enhancement was to make the data more engaging.

4. Reflection

The biggest thing I learned while enhancing this artifact is how quickly performance can be impacted by poorly optimized logic. Without efficient iteration, caching, and modular design, the app began to slow. Keeping everything clean and well-structured helped a lot—especially when linking XML and Java files. I’ve gained a much deeper appreciation for logic reuse, data grouping, and processing loops—concepts I know will benefit future projects.
