Narrative

The artifact that I chose to enhance was a weight-tracking Android mobile application that I had originally worked on in CS-360. This was made using Android Studio and uses a mix of Java and XML files. The purpose of making this application originally was that I was big into working out and eating healthier. This just seemed like a great app to work on out of the options given to us, so I went with it. I still enjoy trying to be healthy, so this app is still very enjoyable to work on. However, since this was my favorite application I have made in school so far, this gave me a great opportunity to work on something that is both fun and gives a lot of potential to show my growth since I started on this about a year and a half ago.

The main reason that I included this artifact for my first enhancement is because it is very diverse with how many tools and imports I am using within it, as well as mobile applications being a career route that I am considering going down. I really enjoyed creating this originally, and backtracking everything and starting from scratch really showed me how much I have improved in my college career. As I mentioned, there are a lot of software development improvements in this new mobile application. I improved the front end overall with a much more polished UI, more pop-up menus or dialogs for cleanliness, dynamic options like timestamps, better SMS handling, more modular code, and overall, a better look to the application.

The difference between this UI/UX and my old project is quite substantial. The code is also of a higher standard, improving all areas of application. I did almost meet all the outcomes that I planned to meet for this objective. The only one that I did not do is setting a personalized icon for the app on the Android home screen. I have been having some struggles with this. Besides that, I went through all the code and rearranged, changed, added or deleted certain classes or XML files with better practice than before. All the classes and files are very simple to understand now with the purpose, where things are inside, as well as better groupings for locations or action controls of the files.

Many files were deleted and then repurposed during this. Now, the classes handle the main login screen, dashboard screen, notes screen and DBHelper (untouched). The XML files in the drawable files handle all icon and text configurations, images, and anything else related to customization. The layout XML files handle the location, colors, and look of everything on each screen. These files are then pulled together by the classes and AndroidManifest.xml to pull everything together. As of now, there are no updates to the outcome-coverage plan.
